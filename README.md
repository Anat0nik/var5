# 🛒 Интернет-магазин: Система учета заказов

## 📋 Вариант 5

### Описание предметной области
Интернет-магазин торгует электроникой. Покупатели оформляют заказы на товары. 
Каждый заказ связывает покупателя с выбранным товаром, фиксирует количество, 
адрес доставки и статус.

## ✨ Реализованные требования

### 2.1 Авторизация
- При запуске отображается окно авторизации
- Данные пользователей хранятся в `users.txt` (формат: login;password;Фамилия Имя)
- Фамилия и имя отображаются в строке состояния

### 2.2 Оконное приложение
- Фиксированный размер: 980×820 пикселей
- Шрифты: Arial (таблицы 9pt, заголовки 14pt полужирный)
- Цвета: фон #FFFFFF, панель вкладок #1A237E, текст вкладок #FFD54F, кнопки #FF5722
- Логотип справа вверху

### 2.3 Справочник «Покупатели»
- Поля: Фамилия, Имя, Email, Телефон, Город
- Контекстное меню: Редактировать, Удалить, Дублировать
- Добавление через кнопку «Добавить»

### 2.4 Справочник «Товары»
- Поля: Наименование, Категория, Цена, Вес
- Категории: Смартфоны, Ноутбуки, Планшеты, Наушники, Аксессуары, Умные часы
- Контекстное меню: Редактировать, Удалить, Дублировать

### 2.5 Сводная сущность «Заказы»
- Поля: ID, Покупатель, Товар, Количество, Адрес доставки, Статус, Сумма
- Статусы: Оформлен, В сборке, В пути, Доставлен, Отменён
- Выбор покупателя и товара из справочников

### 2.6 Модальное окно заказа
- Выпадающие списки покупателей и товаров
- Поле выбора даты
- Автоматический расчет суммы
- Кнопки «Сохранить» и «Отмена»

### 2.7 Работа с CSV
- Сохранение заказов в CSV (Файл → Сохранить как)
- Загрузка заказов из CSV (Файл → Открыть)

### 2.8 Алгебраическая обработка данных
- Сумма = Цена × Количество + 500₽ (доставка)
- Отклонение от среднего в отдельной колонке

### 2.9 Валидация данных
- Email: содержит @ и точку
- Телефон: 11 цифр, первая 8
- Количество: целое число от 1 до 100
